# 鸣潮伤害计算器

> 独立的《鸣潮》(Wuthering Waves) 伤害计算器。纯前端静态页面，浏览器打开即用，无需构建、无后端。

在线使用：[GitHub Pages](https://chuan-hane.github.io/wuwa-damage-calculator/) / [Cloudflare Workers](https://wuwa-damage-calculator.chuan-hane.workers.dev)

项目尽可能模拟真实游戏结算：角色基础面板、属性树、武器、声骸、共鸣链和 Buff 都结构化录入，结算页按当前技能、队伍、链数、延奏、资源、状态和前置自动判断本次攻击能吃到什么。

---

## 特性

- **角色面板挂载**：角色基础攻防生命、双暴、属性树固定加成、武器数值由结构化数据提供；声骸词条可按游戏「声骸总面板」录入，也可由详细声骸模式自动汇总。
- **真实前置判定**：Buff 不因为“角色拥有”就默认生效；必须满足技能、状态、资源、层数、队伍位置、延奏、触发动作等前置才会自动应用。
- **自动 Buff 结算**：本次攻击可触发的 Buff 会在结算区按提供方展示；需要确认的前置可手动勾选，不满足门槛的 Buff 折叠显示。
- **形态分支隔离**：双模态、多形态或阶段角色以当前选择为准；技能、Buff、偏移/效应相关项只显示当前分支和公共项。
- **效应独立结算**：效应区可选择效应提供者；攻击型效应按提供者面板攻击计算，虚湮按层数降低防御并计入普通伤害防御系数。
- **可变 Buff 数值**：支持叠层，也支持按提供者自身属性换算数值，例如守岸人按自身共鸣效率换算团队暴击/暴击伤害。
- **乘区严格分离**：加成区、加深区、技能倍率提升、倍率直接增加、暴击、防御、抗性、最终伤害提升等乘区分开计算。
- **多段技能展示**：技能倍率和伤害可以显示为 `xxx + xxx × N` 形式；最终伤害和多段伤害按向下取整显示。
- **装备系统**：所有5星及4星武器支持谐振阶 1-5 及武器技能效果；角色共鸣链 0-6；声骸按角色默认套装固定展示，支持 5 件单套、3+2、1+2+2、首位声骸和套装/单体效果，也可以自定义。
- **详细声骸模式**：队伍卡可开启详细声骸模式；开启后角色面板的声骸加成输入改为只读展示，由下方「声骸详情」的主词条、副词条档位、套装和首位效果实时汇总。

## 计算口径

### 面板

攻击/生命/防御先算面板，再参与伤害：

```text
攻击 = (角色攻击 + 武器攻击) × (1 + 攻击%总和) + 声骸固定攻击
生命 = 角色生命 × (1 + 生命%总和) + 声骸固定生命
防御 = 角色防御 × (1 + 防御%总和) + 声骸固定防御
```

其中攻击%、生命%、防御%包含属性树、武器、Buff 和声骸百分比词条。声骸固定值加在百分比乘法之外。

声骸有两种录入方式：

- 常规模式：在角色面板 ECHO 行录入声骸总面板词条增益，空值按 0 处理。
- 详细声骸模式：队伍卡开启后，角色面板 ECHO 行只读；「声骸详情」独立成卡并显示当前角色，第一只声骸选择首位且选项自带 Cost（如 `哀声鸷·4c`），其余声骸选择 Cost、有效主词条和副词条档位，合计 Cost 上限 12。

### 伤害

```text
最终伤害 =
  属性基数 × 技能倍率
  × (1 + 加成区)
  × (1 + 加深区)
  × 伤害减免区
  × 防御系数
  × 抗性系数
  × (1 + 最终伤害提升)
```

### 效应伤害

```text
A 类固定值效应伤害 =
  层数基础值
  × (1 + 效应加深)
  × 防御系数
  × 抗性系数

B 类攻击效应伤害 =
  效应提供者攻击 × 层数倍率
  × (1 + 效应加深)
  × 防御系数
  × 抗性系数
```

### 偏移体系

偏移体系按目标 Cost 选择谐度基础值：`1C=716`、`3C=2149`、`4C=10027`。

```text
谐度破坏伤害 =
  谐度基础值 × 1600%
  × (1 + 谐度破坏增幅)
  × 防御系数
  × 伤害减免区
  × (1 + 最终伤害提升)
  × 0.8

骇破伤害/震谐伤害 =
  谐度基础值 × 骇破倍率/震谐倍率
  × (1 + 谐度破坏增幅)
  × 防御系数
  × 抗性系数
  × 伤害减免区
  × (1 + 最终伤害提升)

集谐干涉收益 =
  层数 × 谐度破坏增幅 × 0.12%
```

取整口径：

- 角色、武器等基础攻防生命按游戏显示整数录入。
- 面板攻击/生命/防御显示时向下取整。
- 伤害计算使用显示取整前的面板攻击/生命/防御。
- 技能等级折算后的倍率保留小数点后两位。
- 伤害链各乘区连乘，中途不逐步取整。
- 最终的伤害数值使用 `Math.floor` 向下取整。

例：

```text
基础数值（角色+武器）： 412 + 587 = 999
带攻击加成的最终攻击面板：(412 + 587) × (1 + 12% + 88.1%) + 440 = 2438.999
在游戏中显示的数值向下取整：floor(2438.999) = 2438

伤害计算用 2438.999，最终伤害向下取整：
floor(2438.999 × 0.4865 × 1.964 × 0.5107526882 × 0.9 × 2.588) = floor(2772.38405782) = 2772
注：我方90级｜敌方83级｜敌方抗性10%｜伤害加成96.4%｜暴击伤害258.8%｜技能倍率48.65%

```
注：此取整方式不代表官方，可能会有误差。

## 数据说明

- 常规模式下，声骸词条输入采用声骸总面板词条增益，主副词条已合计，不在计算器里重新拆主副词条。
- 详细声骸模式下，「声骸详情」按游戏内档位拆主词条和副词条并实时汇总到角色面板；主词条只允许选择该 Cost 下有效的词条，副词条只能选择固定档位。
- 声骸混搭组合按角色默认配置，不提供自由改套装入口；3+2、1+2+2 首位可来自包含的所有套装的首位之一，默认优先选择角色元素匹配项。
- Buff 乘区和真实前置需要结合技能原文结构化录入；不确定前置默认不自动应用。
- Beta 角色、武器、4C 首位声骸和声骸套装会在页面上显示 `BetaX.Y.Z` 标识；beta 标识只说明数据来源版本，不改变计算口径。
- 声骸仅录入 4C 及部分 3C（如此间永驻之光的荣光节使）。
