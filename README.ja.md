# 鳴潮ダメージ計算機

> 独立した『鳴潮』ダメージ計算機です。純粋なフロントエンドの静的ページで、ブラウザでそのまま開いて使えます。ビルド手順もバックエンドも不要です。

オンライン版: [wuwa-damage-calculator.chuan-hane.workers.dev](https://wuwa-damage-calculator.chuan-hane.workers.dev)

このプロジェクトは実際のゲーム内ダメージ計算にできるだけ近い形を目指しています。共鳴者の基礎ステータス、スキルツリー固定ボーナス、武器、音骸、共鳴チェーン、Buffを構造化データとして保持し、計算画面では現在のスキル、チーム、共鳴チェーン段階、終奏スキル、リソース、戦闘状態、その他の前提条件から、今回の攻撃に適用できる効果を判定します。

---

## 機能

- **共鳴者ステータスパネル連携**: 基礎攻撃力/HP/防御力、クリティカル、クリティカルダメージ、スキルツリー固定ボーナス、武器ステータスは構造化データから取得します。音骸ステータスはゲーム内の音骸合計ステータスパネル基準で入力でき、詳細音骸モードでは自動集計できます。
- **実戦前提条件の判定**: Buffは共鳴者が所持しているだけでは自動適用されません。現在のスキル、状態、リソース、スタック数、チーム位置、終奏スキル、発動行動などの前提条件を満たした場合のみ自動適用されます。
- **自動 Buff 計算**: 今回の攻撃で使える Buff を計算エリアに提供者別で表示します。確認が必要な前提条件は手動で切り替えられ、条件を満たさない Buff は非アクティブ欄に折りたたまれます。
- **分岐の隔離**: 二重モード、形態、段階を持つ共鳴者は現在の選択を基準にします。スキル、Buff、協和失調/異常効果の項目は、現在の分岐と共通項目のみ表示します。
- **異常効果の独立計算**: 異常効果エリアでは異常効果の提供者を選択できます。攻撃力参照の異常効果は提供者のパネル攻撃力で計算し、虚滅効果はスタック数に応じた防御力ダウンとして通常ダメージの防御係数に反映します。
- **可変 Buff 値**: スタック型 Buff と、提供者自身のステータスに応じて換算される Buff に対応します。例えばショアキーパーは自身の共鳴効率をチームのクリティカル/クリティカルダメージへ換算します。
- **乗区の厳密な分離**: ダメージアップ、ダメージブースト、ダメージ倍率アップ、倍率の直接加算、クリティカル、防御、耐性、最終ダメージアップなどを分けて計算します。
- **多段スキル表示**: スキル倍率とダメージを `xxx + xxx × N` 形式で表示できます。最終ダメージと多段ダメージは切り捨てて表示します。
- **装備システム**: すべての星5・星4武器はランク1-5と武器効果に対応します。共鳴チェーンは0-6段階に対応します。音骸は各共鳴者のデフォルト音骸セットを使い、5セット、3+2、1+2+2、メイン音骸、セット/単体音骸効果を含みます。
- **詳細音骸モード**: チームカードから詳細音骸モードを有効化できます。有効化後、共鳴者パネルの音骸ステータス入力は読み取り専用になり、別カードの音骸詳細にあるメインステータス、サブステータス段階、セット、メイン音骸効果がリアルタイムで合算されます。

## 計算モデル

### パネル

攻撃力、HP、防御力をまずパネルステータスとして計算し、その後ダメージ計算に使います。

```text
攻撃力 = (共鳴者攻撃力 + 武器攻撃力) × (1 + 攻撃力%合計) + 固定音骸攻撃力
HP     = 共鳴者HP × (1 + HP%合計) + 固定音骸HP
防御力 = 共鳴者防御力 × (1 + 防御力%合計) + 固定音骸防御力
```

攻撃力%、HP%、防御力%にはスキルツリー固定ボーナス、武器、Buff、音骸の割合ステータスが含まれます。音骸の固定値は割合乗算の外側で加算します。

音骸ステータスは2通りで入力できます。

- 通常モード: 共鳴者パネルの ECHO 行に音骸合計ステータス上昇値を入力します。空欄は0として扱います。
- 詳細音骸モード: チームカードから有効化すると、共鳴者パネルの ECHO 行は読み取り専用になります。音骸詳細は現在の共鳴者用の別カードとして表示されます。1体目の音骸は `哀切の凶鳥・4c` のように Cost 付きのメイン音骸を選択し、残りの音骸は Cost、有効なメインステータス、サブステータス段階を選択します。合計 Cost 上限は12です。

### ダメージ

```text
最終ダメージ =
  ステータス基礎値 × スキル倍率
  × (1 + ダメージアップ乗区)
  × (1 + ダメージブースト乗区)
  × ダメージ軽減 / 被ダメージアップ乗区
  × 防御係数
  × 耐性係数
  × (1 + 最終ダメージアップ)
```

### 異常効果ダメージ

```text
A類固定値異常効果ダメージ =
  スタック数基礎値
  × (1 + 異常効果ダメージブースト)
  × 防御係数
  × 耐性係数

B類攻撃力参照異常効果ダメージ =
  異常効果提供者攻撃力 × スタック倍率
  × (1 + 異常効果ダメージブースト)
  × 防御係数
  × 耐性係数
```

異常効果ダメージはクリティカル、クリティカルダメージ、属性ダメージアップ、タイプダメージアップ、通常のダメージブースト、最終ダメージアップの影響を受けません。虚滅効果は単独ダメージではなく、スタック数に応じた防御力ダウンとして扱われ、選択時に通常ダメージの防御係数へ反映されます。

### 協和失調システム

協和失調システムでは、目標 Cost に応じて協和基礎値を選択します: `1C=716`、`3C=2149`、`4C=10027`。

```text
協和破壊ダメージ =
  協和基礎値 × 1600%
  × (1 + 協和破壊ブースト)
  × 防御係数
  × ダメージ軽減 / 被ダメージアップ乗区
  × (1 + 最終ダメージアップ)
  × 0.8

ハックダメージ / 震撃協和ダメージ =
  協和基礎値 × ハック倍率 / 震撃協和倍率
  × (1 + 協和破壊ブースト)
  × 防御係数
  × 耐性係数
  × ダメージ軽減 / 被ダメージアップ乗区
  × (1 + 最終ダメージアップ)

密集協和・インターフェア効果量 =
  スタック数 × 協和破壊ブースト × 0.12%
```

切り捨て規則:

- 共鳴者と武器の基礎攻撃力/HP/防御力は、ゲーム表示の整数値で入力します。
- 表示されるパネル攻撃力/HP/防御力は切り捨てます。
- ダメージ計算には表示前の、切り捨てていないパネル攻撃力/HP/防御力を使います。
- スキルレベル補正後のスキル倍率は小数第2位まで保持します。
- ダメージ乗区は途中で切り捨てず、連続して乗算します。
- 最終表示ダメージは `Math.floor` を使用します。

例:

```text
基礎値（共鳴者 + 武器）: 412 + 587 = 999
攻撃力ボーナス適用後の最終攻撃力パネル: (412 + 587) × (1 + 12% + 88.1%) + 440 = 2438.999
ゲーム表示値の切り捨て: floor(2438.999) = 2438

ダメージ計算には 2438.999 を使い、最終ダメージは切り捨てます。
floor(2438.999 × 0.4865 × 1.964 × 0.5107526882 × 0.9 × 2.588) = floor(2772.38405782) = 2772
注: 味方共鳴者 Lv.90 | 敵 Lv.83 | 敵耐性10% | ダメージアップ96.4% | クリティカルダメージ258.8% | スキル倍率48.65%
```

この切り捨てモデルは公式のものではなく、ゲーム内結果と差が出る場合があります。

## データメモ

- 通常モードでは、音骸ステータス入力は音骸合計ステータス上昇値を使います。メインステータスとサブステータスはすでに合算済みとして扱い、計算機内では再分解しません。
- 詳細音骸モードでは、音骸詳細がゲーム内の段階値に基づいてメインステータスとサブステータスを分解し、共鳴者パネルへリアルタイムに合算します。メインステータスは選択した Cost で有効なステータスに限られ、サブステータスは固定段階値のみ使用できます。
- 音骸の混合セットは各共鳴者のデフォルト構成に従い、自由なセット編集は提供しません。3+2 と 1+2+2 構成では、メイン音骸は含まれるセットのメイン音骸から選べ、デフォルトでは共鳴者の属性に合う項目を優先します。
- Buff の乗区と実戦前提条件は、スキルテキストを構造化して入力する必要があります。不確かな前提条件はデフォルトで自動適用しません。
- 音骸データは現在 4C 音骸と一部の 3C 音骸を含みます。例: エターナル・ライトのカピタネウス。
