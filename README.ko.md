# 명조 피해 계산기

> 독립적인 《명조》 피해 계산기입니다. 순수 프론트엔드 정적 페이지라 브라우저에서 바로 열어 사용할 수 있으며, 빌드 단계와 백엔드가 필요하지 않습니다.

온라인 사용: [wuwa-damage-calculator.chuan-hane.workers.dev](https://wuwa-damage-calculator.chuan-hane.workers.dev)

이 프로젝트는 실제 게임 피해 계산을 최대한 가깝게 모델링합니다. 공명자 기본 스탯, 스킬 트리 고정 보너스, 무기, 에코, 공명 체인, 버프를 구조화 데이터로 보관하고, 계산 화면은 현재 스킬, 파티, 공명 체인 단계, 반주 스킬, 자원, 전투 상태와 기타 선행 조건에 따라 이번 공격이 받을 수 있는 효과를 판정합니다.

---

## 기능

- **공명자 스탯 패널 연결**: 기본 공격력/HP/방어력, 크리티컬 확률, 크리티컬 피해, 스킬 트리 고정 보너스, 무기 수치는 구조화 데이터에서 가져옵니다. 에코 스탯은 게임 내 에코 총 스탯 패널 기준으로 입력하거나, 상세 에코 모드에서 자동 합산할 수 있습니다.
- **실제 선행 조건 판정**: 버프는 공명자가 보유했다는 이유만으로 자동 적용되지 않습니다. 현재 스킬, 상태, 자원, 스택 수, 파티 위치, 반주 스킬, 발동 행동 등 선행 조건을 만족해야 자동 적용됩니다.
- **자동 버프 계산**: 이번 공격에 적용될 수 있는 버프를 계산 영역에서 제공자별로 표시합니다. 확인이 필요한 선행 조건은 수동으로 켤 수 있고, 조건을 만족하지 못한 버프는 비활성 섹션에 접어 표시합니다.
- **분기 격리**: 이중 모드, 형태, 단계가 있는 공명자는 현재 선택을 기준으로 합니다. 스킬, 버프, 편조/이상 효과 항목은 현재 분기와 공통 항목만 표시합니다.
- **이상 효과 독립 계산**: 이상 효과 영역에서 이상 효과 제공자를 선택할 수 있습니다. 공격력 기반 이상 효과는 제공자의 패널 공격력으로 계산하며, 인멸 재앙은 스택 수에 따른 방어 감소로 처리해 일반 피해 방어 계수에 반영합니다.
- **가변 버프 수치**: 스택형 버프와 제공자 자신의 스탯에 따라 환산되는 버프를 지원합니다. 예를 들어 파수인은 자신의 공명 효율을 팀 크리티컬 확률/크리티컬 피해로 환산합니다.
- **곱연산 구역 분리**: 피해 보너스, 피해 심화, 피해 배율 증가, 배율 직접 증가, 크리티컬, 방어, 내성, 최종 피해 보너스 등을 분리해 계산합니다.
- **다단 스킬 표시**: 스킬 배율과 피해를 `xxx + xxx × N` 형식으로 표시할 수 있습니다. 최종 피해와 다단 피해는 내림 처리해 표시합니다.
- **장비 시스템**: 모든 5성 및 4성 무기는 랭크 1-5와 무기 효과를 지원합니다. 공명 체인은 0-6단계를 지원합니다. 에코는 각 공명자의 기본 세트 구성을 사용하며, 5세트, 3+2, 1+2+2, 메인 에코, 세트/단일 에코 효과를 포함합니다.
- **상세 에코 모드**: 파티 카드에서 상세 에코 모드를 켤 수 있습니다. 켜면 공명자 패널의 에코 스탯 입력이 읽기 전용이 되고, 별도 에코 상세 카드의 메인 스탯, 부옵션 등급, 세트, 메인 에코 효과가 실시간 합산됩니다.

## 계산 모델

### 패널

공격력, HP, 방어력을 먼저 패널 스탯으로 계산한 뒤 피해 계산에 사용합니다.

```text
공격력 = (공명자 공격력 + 무기 공격력) × (1 + 총 공격력%) + 고정 에코 공격력
HP     = 공명자 HP × (1 + 총 HP%) + 고정 에코 HP
방어력 = 공명자 방어력 × (1 + 총 방어력%) + 고정 에코 방어력
```

공격력%, HP%, 방어력%에는 스킬 트리 고정 보너스, 무기, 버프, 에코 퍼센트 스탯이 포함됩니다. 에코 고정값은 퍼센트 곱연산 밖에서 더합니다.

에코 스탯은 두 가지 방식으로 입력할 수 있습니다.

- 일반 모드: 공명자 패널의 ECHO 행에 에코 총 스탯 증가량을 입력합니다. 빈 값은 0으로 처리합니다.
- 상세 에코 모드: 파티 카드에서 켜면 공명자 패널의 ECHO 행이 읽기 전용이 됩니다. 에코 상세는 현재 공명자의 별도 카드로 표시됩니다. 첫 번째 에코는 `애곡하는 아익스·4c`처럼 Cost가 포함된 메인 에코를 선택하고, 나머지 에코는 Cost, 유효 메인 스탯, 부옵션 등급을 선택합니다. 총 Cost 상한은 12입니다.

### 피해

```text
최종 피해 =
  스탯 기준값 × 스킬 배율
  × (1 + 피해 보너스 구역)
  × (1 + 피해 심화 구역)
  × 피해 감소 / 취약 구역
  × 방어 계수
  × 내성 계수
  × (1 + 최종 피해 보너스)
```

### 이상 효과 피해

```text
A류 고정값 이상 효과 피해 =
  스택 수 기준값
  × (1 + 이상 효과 피해 심화)
  × 방어 계수
  × 내성 계수

B류 공격력 기반 이상 효과 피해 =
  이상 효과 제공자 공격력 × 스택 배율
  × (1 + 이상 효과 피해 심화)
  × 방어 계수
  × 내성 계수
```

이상 효과 피해는 크리티컬 확률, 크리티컬 피해, 속성 피해 보너스, 유형 피해 보너스, 일반 피해 심화, 최종 피해 보너스를 받지 않습니다. 인멸 재앙은 독립 피해가 아니며, 스택 수에 따른 방어 감소로 처리되어 선택 시 일반 피해 방어 계수에 포함됩니다.

### 편조 시스템

편조 시스템은 목표 Cost에 따라 조화도 기준값을 선택합니다: `1C=716`, `3C=2149`, `4C=10027`.

```text
조화도 파괴 피해 =
  조화도 기준값 × 1600%
  × (1 + 조화도 파괴 증폭)
  × 방어 계수
  × 피해 감소 / 취약 구역
  × (1 + 최종 피해 보너스)
  × 0.8

해킹 파괴 피해 / 진동 파열 피해 =
  조화도 기준값 × 해킹 파괴 배율 / 진동 파열 배율
  × (1 + 조화도 파괴 증폭)
  × 방어 계수
  × 내성 계수
  × 피해 감소 / 취약 구역
  × (1 + 최종 피해 보너스)

조화 밀집 · 간섭 수익 =
  스택 수 × 조화도 파괴 증폭 × 0.12%
```

내림 규칙:

- 공명자와 무기의 기본 공격력/HP/방어력은 게임에 표시되는 정수값으로 입력합니다.
- 표시되는 패널 공격력/HP/방어력은 내림 처리합니다.
- 피해 계산에는 표시 전의 내림 처리하지 않은 패널 공격력/HP/방어력을 사용합니다.
- 스킬 레벨 보정 후 스킬 배율은 소수점 둘째 자리까지 유지합니다.
- 피해 구역은 중간 내림 없이 연속으로 곱합니다.
- 최종 표시 피해값은 `Math.floor`를 사용합니다.

예시:

```text
기본값(공명자 + 무기): 412 + 587 = 999
공격력 보너스 적용 후 최종 공격력 패널: (412 + 587) × (1 + 12% + 88.1%) + 440 = 2438.999
게임 표시값 내림: floor(2438.999) = 2438

피해 계산은 2438.999를 사용하고, 최종 피해는 내림 처리합니다.
floor(2438.999 × 0.4865 × 1.964 × 0.5107526882 × 0.9 × 2.588) = floor(2772.38405782) = 2772
메모: 아군 공명자 Lv.90 | 적 Lv.83 | 적 내성 10% | 피해 보너스 96.4% | 크리티컬 피해 258.8% | 스킬 배율 48.65%
```

이 내림 모델은 공식 모델이 아니며, 게임과 차이가 있을 수 있습니다.

## 데이터 메모

- 일반 모드에서 에코 스탯 입력은 에코 총 스탯 증가량을 사용합니다. 메인 스탯과 부옵션은 이미 합산된 값으로 보고, 계산기에서 다시 분해하지 않습니다.
- 상세 에코 모드에서 에코 상세는 게임 내 등급값 기준으로 메인 스탯과 부옵션을 분해해 공명자 패널에 실시간 합산합니다. 메인 스탯은 선택한 Cost에서 가능한 스탯으로 제한되고, 부옵션은 고정 등급값만 사용할 수 있습니다.
- 에코 혼합 세트는 각 공명자의 기본 구성을 따르며, 자유 세트 편집은 제공하지 않습니다. 3+2와 1+2+2 구성에서 메인 에코는 포함된 세트의 메인 에코 중 하나를 사용할 수 있고, 기본값은 공명자 속성과 맞는 항목을 우선합니다.
- 버프 곱연산 구역과 실제 선행 조건은 스킬 텍스트를 구조화해 입력해야 합니다. 불확실한 선행 조건은 기본적으로 자동 적용하지 않습니다.
- Beta 공명자, 무기, 4C 메인 에코, 에코 세트는 페이지에 `BetaX.Y.Z` 표시가 붙습니다. 이 표시는 데이터 출처 버전만 나타내며 계산 규칙을 바꾸지 않습니다.
- 에코 데이터는 현재 4C 에코와 일부 3C 에코를 포함합니다. 예: 영원의 광채의 카피타네우스.
